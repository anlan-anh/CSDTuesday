General Trees - Definitions

Definition: A tree is an abstract data type storing elements hierarchically.

Nodes have a parent-child relationship, except for the root (no parent).

Reference: Pages 308-311

General Trees - Key Terms (Part 1)

Key Terms:

  • Root: Node without a parent.
  • Leaf (External Node): Node without children.
  • Internal Node: Node with at least one child.
  • Edges: Connect parent to child.
Reference: Pages 308-311

General Trees - Key Terms (Part 2) & Diagram

  • Ancestors/Descendants: Define lineage.
  • Depth: Number of ancestors (root depth = 0).
  • Height: Longest path from a node to a leaf; tree height is root's height.

Binary Trees (BT) - Definitions & Properties

Definition: An ordered tree where each node has at most two children: a left child and a right child.

Left child precedes the right child.

Properties:

  • Proper (Full) BT: Each node has 0 or 2 children.
  • Max nodes at level d is 2ᵈ.
  • Min height for n nodes is Ω(log n) (specifically, height is at least ⌈log₂(n+1)⌉-1 for n nodes or log₂ n if leaves are at level h and n is number of leaves for a proper BT).

Binary Search Trees (BST) - Definition

Definition: A binary tree where each node p stores a key (and optionally a value) such that:

  • Keys in the left subtree of p are less than p's key.
  • Keys in the right subtree of p are greater than p's key.

Advantages:

  • Efficient search, insertion, deletion (average O(log n) if balanced).
  • Maintains element order.

BST - Searching

Algorithm:

  • Start at the root.
  • Compare target key k with current node's key p.key():
    • If k < p.key(), search left subtree.
    • If k == p.key(), key found.
    • If k > p.key(), search right subtree.
  • If a null subtree is reached, key not found.

Performance: O(h), where h is height. O(log n) for balanced, O(n) for skewed.

Reference: Pages 461-462

BST - Node Insertion

Algorithm:

  • To insert key k (and associated value):
  • Search for key k to find the leaf (or external node z) where k should be.
  • If k is found at an existing node, policy for duplicates applies (e.g., update value, or as per book's approach, expand the node).
  • If k is not found, add the new node as a child of the parent of z (the last internal node encountered during search).
  • New nodes are always inserted effectively as new leaves.

Performance: O(h).

Reference: Pages 463-465

  • References: parent, left child, right child.

BST - Implementation Overview (Java)

Class TreeMap (Book's Example for BST logic):

  • Uses a nested Node class.
  • Maintains root reference and size.
  • The book's TreeMap uses an inner BalanceableBinaryTree class to encapsulate tree operations like insertions and rotations.

Core Methods (Conceptual):

  • get(k): Searches for key k.
  • put(k, v): Inserts key-value pair (k,v).
